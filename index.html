Core Loop

Fixed-Timestep Game Loop

The game loop runs on a dedicated thread separate from Swing's EDT. An accumulator-based fixed-timestep at 60 FPS ensures deterministic physics regardless of rendering speed — critical for the lockstep multiplayer model.

Mouse coordinates are back-projected through the scale/offset affine transform to the logical 1280×720 space before any game logic reads them, decoupling display resolution from simulation.

OOP Design

Design Patterns Used

The codebase applies six classical patterns deliberately, each chosen for a specific subsystem need — not for completeness.

Object Pool State Machine Strategy Observer-lite Factory Template Method

Projectiles use an Object Pool of 256 slots — zero allocation during live combat, eliminating GC pauses. Weapon types are a Strategy enum. Maps are built by MapFactory. Kill events propagate via lightweight queues (Observer-lite) to the HUD and KillFeed.

Physics

PhysicsBody & Collision Resolution

Every movable entity extends PhysicsBody.java. Forces accumulate per frame and integrate via semi-implicit Euler. Collision uses AABB overlap with axis-separated response — horizontal and vertical resolved independently to prevent corner-catching artifacts.

Projectile behaviour is parameterised: bullets carry gravScale=0 for straight flight; grenades use full gravity with a bounce timer; rockets store velocity magnitude for rotation rendering. The double-jump counter (jumpsAvailable) resets on landing and uses edge-triggered key detection via InputState.pressed() — each key-down fires exactly once regardless of hold duration.

Artificial Intelligence

BotController State Machine

Bot AI is a five-state machine implemented in BotController.java:

PATROL → CHASE → ATTACK → SEEK_HEALTH → FLEE

Bots use ray-cast line-of-sight checks against tile geometry, remember last-seen target positions when LOS is broken, and use lead-target aiming — calculating where the player will be at bullet travel time. Retreat triggers at <12% HP. Difficulty scales two parameters: reaction delay and aim error radius, producing three distinct difficulty tiers without branching logic.

Networking

Relay-Based Lockstep Multiplayer

Online play uses a lockstep input model over plain TCP sockets — no third-party networking library. One player creates a lobby and receives a 4-letter code; others join by code. Each client sends its raw input state for the current frame to RelayServer.java, which broadcasts all inputs to all peers. Every client then runs an identical physics simulation from the same inputs, keeping game state in sync without sending world snapshots.

The fixed-timestep loop introduces a deliberate 1–2 frame input delay, which buffers against network jitter and makes the game playable up to ~100ms RTT — typical India-to-India latency of 20–40ms is well within this window. RelayServer.java is self-contained with zero external dependencies and can be deployed to any free TCP hosting service (Railway, Render). Host/port are configured via GameConstants.java.
